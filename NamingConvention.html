Tools

Unfortunatly, with Unreal being so restrictive with it's files we have to use tools, Fortunatly i like to make tools and make them accessible for every discipline

The 2 tools that we will be using is, Github Desktop and GitLFS Locking App. Both of these application are easy to use and i will explain more in detail how you should use them, and what to do if an error popup reveals itself, In and .

Work Process

Let's follow a standard day and see how you would go about using the tools to maximize efficiency in teamwork. In the morning the first thing you do is Fetch from the current development branch which is: Alpha. After we have had our standup meeting and you have decided what you are gonna do, there 2 ways forward.

(1) If you have to add a file there is no need to Lock that file

(2) If you have to edit a file you have to Lock that file

When you think you are done for the time being Push your Change and if you are done with editing the file for today Unlock the file so that fellow developers can edit that file later down the line.

Code Conventions

The Coding conventions are split into 2 categories, Blueprint And C++. While C++ is more for programmers, both designers and programmers should make us of these conventions so that no matter at what stage the project is, we can always understand what is put down.

Blueprint Conventions

Before we get to the big stuff, here are some quick blueprint tips.

Remember to Comment sections of your code, if you select multiple functions then press "C" to create a comment block around the selected functions

Check always if a blueprint is locked, and if not check if you yourself have it locked before working on it

To make things more readable straigthen out pin connection, create pin node's. All these have hotkeys to make this less tedious

C++ Conventions

Here are some quick tips to keep our C++ files consistent between people, or atleast, the headers.

We will keep comments in the header file only, make header files very convenient and cpp files less cluttered

Comment as concise but correctly what a method or variable does/returns

UProperty Cheatsheet

When declaring properties, Property Specifiers can be added to the declaration to control how the property behaves with various aspects of the Engine and Editor.

It is ordered in most used, pls msg if some have higher priority

Property Tag

Effect

EditAnywhere

Indicates that this property can be edited by property windows, on archetypes and instances. This Specifier is incompatible with any of the the "Visible" Specifiers.

BlueprintReadWrite

This property can be read or written from a Blueprint. This Specifier is incompatible with the BlueprintReadOnly Specifier.

BlueprintReadOnly

This property can be read by Blueprints, but not modified. This Specifier is incompatible with the BlueprintReadWrite Specifier

Category= "TopCategory| SubCategory|..."

Specifies the category of the property when displayed in Blueprint editing tools. Define nested categories using the | operator.

BlueprintAssignable

Usable with Multicast Delegates only. Exposes the property for assigning in Blueprints.

EditDefaultsOnly

Indicates that this property can be edited by property windows, but only on archetypes. This Specifier is incompatible with any of the "Visible" Specifiers.

AdvancedDisplay

The property will be placed in the advanced (dropdown) section of any panel where it appears.

EditInline

Allows the user to edit the properties of the Object referenced by this property within Unreal Editor's property inspector (only useful for Object references, including arrays of Object reference).

EditInstanceOnly

Indicates that this property can be edited by property windows, but only on instances, not on archetypes. This Specifier is incompatible with any of the "Visible" Specifiers.

VisibleAnywhere

Indicates that this property is visible in all property windows, but cannot be edited. This Specifier is incompatible with the "Edit" Specifiers.

VisibleDefaultsOnly

Indicates that this property is only visible in property windows for archetypes, and cannot be edited. This Specifier is incompatible with any of the "Edit" Specifiers.

VisibleInstanceOnly

Indicates that this property is only visible in property windows for instances, not for archetypes, and cannot be edited. This Specifier is incompatible with any of the "Edit" Specifiers.

BlueprintAuthorityOnly

This property must be a Multicast Delegate. In Blueprints, it will only accept events tagged with BlueprintAuthorityOnly.

BlueprintCallable

Multicast Delegates only. Property should be exposed for calling in Blueprint code.

BlueprintGetter= GetterFunctionName

This property specifies a custom accessor function. If this property isn't also tagged with BlueprintSetter or BlueprintReadWrite, then it is implicitly BlueprintReadOnly.

BlueprintSetter= SetterFunctionName

This property has a custom mutator function, and is implicitly tagged with BlueprintReadWrite. Note that the mutator function must be named and part of the same class.

Config

This property will be made configurable. The current value can be saved to the .ini file associated with the class and will be loaded when created. Cannot be given a value in default properties. Implies BlueprintReadOnly.

DuplicateTransient

Indicates that the property's value should be reset to the class default value during any type of duplication (copy/paste, binary duplication, etc.).

EditFixedSize

Only useful for dynamic arrays. This will prevent the user from changing the length of an array via the Unreal Editor property window.

UClass Cheatsheet

When declaring classes, Class Specifiers can be added to the declaration to control how the class behaves with various aspects of the Engine and Editor.

It is ordered in most used, pls msg if some have higher priority

Class Specifier

Effect

Blueprintable

Exposes this class as an acceptable base class for creating Blueprints. The default is NotBlueprintable, unless inherited otherwise. This Specifier is inherited by subclasses.

NotBlueprintable

Specifies that this class is not an acceptable base class for creating Blueprints. This is the default and is inherited by subclasses.

NotPlaceable

Negates a Placeable Specifier inherited from a base class. Indicates that Objects of this class may not be placed into a Level, UI scene, or Blueprint in the Editor.

BlueprintType

Exposes this class as a type that can be used for variables in Blueprints.

Placeable

Indicates that this class can be created in the Editor and placed into a level, UI scene, or Blueprint (depending on the class type). This flag is propagated to all child classes; child classes can override this flag using the NotPlaceable Specifier.

EditInlineNew

Indicates that Objects of this class can be created from the Unreal Editor Property window, as opposed to being referenced from an existing Asset. The default behavior is that only references to existing Objects may be assigned through the Property window). This Specifier is propagated to all child classes; child classes can override this with the NotEditInlineNew Specifier.
